What's New
Version History
New features, enhancements and bug fixes.
Version 3.3.10
What's NewNew Feature
The OANet navigation bar at the top of the screen now includes an icon to access “What’s New”, an update center designed to dynamically inform users of changes to OANet and summarize the contents of this document. Clicking the megaphone icon launches the “What’s New” popup where the user can watch videos or read short summaries of all recent updates, enhancements, and bug fixes in OANet.
People Seeds and Client SeedsNew Feature
In the December OANet update, a new concept was been added to OANet: Seeds. Users can designate People records as People Seeds, and Client records as Client Seeds – in order to designate a People record as a People Seed, users must change the People Origination Source to “People Seeds”; similarly, to designate a Client record as a Client Seed, users must change the Client Source to “Client Seeds”.
PV Code ConsolidationEnhancement
As a part of new sales initiative for 2020, the PV codes within OANet have been overhauled and a number of codes consolidated.
Mass E-SignatureEnhancement
It is now possible to send a single E-Signature document to multiple recipients using OANet. Launching Mass E-Signature Request is done from the Tools tab.
Credit ScreenEnhancement
The Credit Screen has now been enhanced in order to streamline the process of obtaining credit approval. Users can now enter Direct Hire Fee for clients that are expected to have Direct Hire Placements – the screen now allows AEs to easily input Temp/Temp-to-Hire information or Direct Hire information in the Placement Information section of the screen. The whole screen has furthermore been reorganized to make the process more intuitive.
Client Special BillEnhancement
The Client Special Bill screens and process have been redesigned to be more intuitive and straightforward. While the screens still perform the same functions, the process has now been broken down into a step-by-step sequence to avoid confusion.
Show “Report To” in Search resultsEnhancement
When conducting a Job Order or Placement Search, a new column is now available indicating the “Report To” contact for the associated record.
Worker’s Comp Approval without Credit InformationEnhancement
Previously, it was not possible to generate Worker’s Comp approval in the client screen without first creating a credit record. Itwill now be possible to obtain Worker’s Comp approval without first providing credit information.
Mapped Client (Canada)Enhancement
When mapping Job Orders between databases, the Job Order must have a matching Client within the same database, or else such a Client Record must be created. For offices within Canada, a mapped Job Order will carry the same Worker Comp codes under the following conditions: a) If the target database does not have a Client Record that matches the Job Order being mapped – in this instance, a matching Client Record will be created within the target database with the same Worker Comp codes (mapped Job Order will inherit these codes) b) If the target database has a Client Record that matches the Client Record of the original Job Order and has the same Worker Comp codes, then mapped Job Order will also have these Worker Comp codes In instances where a Job Order is mapped and the target database has a matching Client Record but with different Worker Comp codes, then the Worker Comp section of the mapped Job Order will be blank in the target database.
Recent RecordsBug Fix
Several changes have been made to streamline the use of the Recent Records functionality. Opening the popup is now done by double clicking the icon; additionally, clicking a listed Recent Record will now display record detail on the same line as the record clicked.
People Search Bug FixBug Fix
We've made some enhancements and fixed a few bugs to make OAnet even better for you, because we believe small change adds up. The search results will now be more exact and targetted.
